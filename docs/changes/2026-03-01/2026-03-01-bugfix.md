## 问题概述

代码审查发现以下 lint 错误需要修复：

| 错误类型 | 数量 | 描述 |
|---------|------|------|
| F821 | 2处 | 未定义的名称（类型提示导入缺失） |

## 详细修复

### 1. F821 - `nanobot/agent/loop.py`

**问题**: `ToolsConfig` 类型提示在函数签名中使用，但未在 TYPE_CHECKING 块中导入。

**修复**: 在 TYPE_CHECKING 块添加 `ToolsConfig` 导入。

```python
# 修复前
if TYPE_CHECKING:
    from nanobot.config.schema import ChannelsConfig, ExecToolConfig, RAGConfig
    from nanobot.cron.service import CronService

# 修复后
if TYPE_CHECKING:
    from nanobot.config.schema import ChannelsConfig, ExecToolConfig, RAGConfig, ToolsConfig
    from nanobot.cron.service import CronService
```

### 2. F821 - `nanobot/agent/subagent.py`

**问题**:
- 缺少 `from __future__ import annotations`，导致需要字符串类型提示
- `ExecToolConfig` 类型提示未导入

**修复**:
1. 添加 `from __future__ import annotations`（文件顶部）
2. 添加 TYPE_CHECKING 到 typing 导入
3. 添加 TYPE_CHECKING 块导入 ExecToolConfig
4. 移除类型提示的字符串引号

```python
# 修复前
"""Subagent manager for background task execution."""

import asyncio
...
from typing import Any
...
    exec_config: "ExecToolConfig | None" = None,

# 修复后
"""Subagent manager for background task execution."""

from __future__ import annotations

import asyncio
...
from typing import TYPE_CHECKING, Any
...
if TYPE_CHECKING:
    from nanobot.config.schema import ExecToolConfig
...
    exec_config: ExecToolConfig | None = None,
```

### 3. W293 - 空白行空白字符清理

**修复**: 使用 ruff 自动修复 46 处空白行空白字符问题。

```bash
ruff check --fix --unsafe-fixes nanobot/ --select=W293
```
